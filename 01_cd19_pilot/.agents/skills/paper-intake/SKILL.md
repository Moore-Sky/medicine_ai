# 目标

将一篇论文压缩为可直接支持 lead rescue 决策的标准卡片。

# 输入

- 论文来源：`references/paper_list.md` 中指定条目
- 输出路径：通常为 `cases/case_001_fmc63/intake.md`
- 可选结构记录：PDB 或 RCSB 链接

# 输出

按 `templates/paper_card.md` 产出，必须包含：

1. 引用信息
2. 核心问题
3. 研究系统（target / binder / assay / model）
4. 关键结构发现
5. 关键功能发现
6. 工程改动
7. 对 CD19 lead rescue 的可复用启示
8. 证据与推断拆分（直接证据 / 强推断 / 开放问题）

# 硬约束

- 固定范围：CD19 + FMC63 语境；不扩展到其他靶点。
- 语言：正文用中文；论文标题保持英文原文。
- 只保留对工程决策有用的信息，不写大段背景教学。
- 明确标注哪些结论来自结构证据、功能证据、解释性推断。

# 执行步骤

1. 读取论文标题、摘要和可访问全文。
2. 抽取系统边界：构型、表位/几何、affinity、antigen density、功能 readout。
3. 用模板落盘到指定案例文件。
4. 自检后再保存：
- 是否把推断写成结论
- 是否遗漏风险维度（至少提及 affinity 之外的一个风险）
- 是否出现越界内容（pMHC、湿实验细节承诺等）

# 禁止事项

- 不要一次处理多篇论文并拼接成综述。
- 不要把论文未报告的数据写成“已证明”。
- 不要生成与当前案例无关的大段免疫学解释。
