# 科普: 生物-医学

# 1. 目标: 为 非生物背景的程序员 建立 基础科普路线

无需像医学生系统理解，至少应做到：

- 听到下面概念, 不再完全陌生
  - `细胞膜 / 线粒体 / DNA / RNA / 基因表达 / 染色体异常 / 表观遗传 / 蛋白构象 / 酶 / 代谢 / 炎症 / ADME / 靶点验证`
- 能分清 讨论是在说：
  - **人体功能层**
  - **细胞层**
  - **分子层**
  - **遗传,调控层**
  - **病理层**
  - **药理层**
  - **研发流程层**
- 能问出 靠谱的问题：
  - 这个功能原本是谁负责？
  - 现在最早坏在哪一步？
  - 是 DNA 变了、表达变了，还是蛋白行为变了？
  - 这个药卡的是哪一层？
  - 为什么一个候选看起来能结合，却未必能成药？

# 2. 总顺序

1. 人体整体功能图景
2. 细胞与组织
3. DNA → RNA → 蛋白
4. 遗传 基因科学
5. 蛋白质、生物化学、能量与通路
6. 血液、淋巴、免疫、损伤与修复
7. 病理学主线
8. 药理学主线
9. 药物研发流程常识

# 3. 使用

- 每天 围绕一个小话题, 每天 30 分钟, 每周 5 天
- 每天 框架
  - 词汇：认识什么
  - 结构：怎么分层、摆放、组织。
  - 功能：负责干什么。
  - 过程：怎么一步步发生的。
- 周末 自测
  - 不看笔记，用自己的话讲出本周主线。
  - 给本周每天的话题各写一句最小定义。
  - 随机抽 1 天，把它的过程画成 4~6 步箭头图。
- 回顾 每3周一次
  - 哪些词已经不再陌生？
  - 哪些结构已经能在脑中分层？
  - 哪些功能已经能说出“为什么需要它”？
  - 哪些过程已经能讲成一条完整小链？
