# 免疫治疗项目方向 沟通纪要

这次会的关键结论 不是 “立刻用 AI 设计药”，而是：

> 先成为能看懂免疫靶点、能读结构论文、能辅助候选分析的工程师。

当前最重要的两个目标：

1. 先搞懂全局图景。
2. 再挑一个最简单、最可验证、最适合计算辅助的切口落地。

会议反复强调的节奏是：**先补知识，再上环境**。

## 一、目标

### 1. 总目标

团队聚焦两条药物/免疫工程路线，不是泛泛讨论“AI+生物”，而是聚焦于免疫治疗相关的具体靶点识别与工程化问题。

**路线 A：CD70**

- 采用较传统的 CAR-T / 抗体工程思路。
- 目标是打一个新的表面靶点 CD70。

**路线 B：pMHC / CoStAR**

- 识别肿瘤细胞表面的 pMHC 复合物。
- 目的是间接打到胞内突变抗原（如会议中反复提到的 TP53/P53）。
- 在本次会议的讨论框架中，CoStAR 可暂时理解为：用更高亲和力的识别模块结合 TCR 信号底盘，以增强对低密度 pMHC 靶点的识别与激活能力。

可先粗略记成：

- `CD70 = 老方法打新靶点`
- `pMHC / CoStAR = 新方法打新靶点`

## 二、非生物背景需要先懂的核心常识

以下表述以建立直觉为主，属于工程化入门类比，不等同于严格免疫学定义。

### 1. T 细胞在“巡逻抓坏人”

免疫学背景可简化为：

- 细胞会把蛋白碎片挂到 MHC 上并展示到表面。
- TCR 会识别这些展示内容。
- 自体正常蛋白通常已建立免疫耐受。
- 病毒蛋白或肿瘤突变蛋白可能被识别为异常。

程序员类比：

- 细胞把“内部发生了什么”的摘要贴在门口。
- T 细胞像安检员，发现异常摘要就可能开火。

### 2. pMHC 是“门口贴出来的异常摘要”

本质过程：

1. 某个蛋白被切成小片段。
2. 小片段挂到 MHC/HLA 上。
3. 二者作为复合物出现在细胞表面。

程序员类比：

- 原始蛋白像后端完整日志。
- pMHC 像前端弹出的一行告警摘要。
- 打不到整份内部日志，但可以识别这条告警。

### 3. 肿瘤为什么难打

会议强调的原因：

- 肿瘤突变抗原虽然存在，但表面展示量往往很低。
- 信号强度不足，导致“点不着火”。
- 天然 TCR 或普通抗体常常不够强、不够稳。

可理解为：不是没有 bug，而是日志太少、信号太弱、误报风险高。

### 4. CoStAR 为什么让团队兴奋

会议中的关键观点可工程化理解为：

- 天然 TCR 是一个可用但不一定足够强的识别模块。
- 团队希望把“更强识别头”与“更强激活尾巴”组合。
- 即把高亲和力抗体识别能力接入 TCR 执行底盘。

可暂时理解为：**更强传感器头 + 成熟执行底盘**。

## 三、分配给程序员的真实任务

### 1. 先建立认知，不急着搭环境

会议已明确：

- 近期不着急上环境。
- 先补知识、看综述、看历史案例。
- 先理解别人怎么做，再谈算力与工程化落地。

### 2. 补“识别机制”的结构认知

计算方向重点在：

- 抗体/识别头如何抓住目标。
- 抗体与 pMHC 的结合结构基础。
- 在此基础上再谈亲和力优化与分子改造。

这不是“直接端到端做疗法”，而是先解决“如何识别”。

### 3. 这是干湿结合，不是纯算法

会议明确了后续可能路径：

- 可能外包湿实验。
- 先拿到野生抗体序列。
- 再基于结构或模型做优化。

你的角色更接近：**设计辅助 / 结构分析 / 候选筛选 / 知识整理工程师**。

## 四、可作为项目文档的摘要

### 1. 项目目标

建立面向免疫治疗分子的认知与计算辅助能力，短期围绕两条路线学习拆解：

- `CD70`：传统 CAR-T / 抗体工程，打表面靶点。
- `pMHC / CoStAR`：尝试打胞内突变抗原经 pMHC 呈递后的表面复合物。

### 2. 当前难点

- pMHC 靶点展示量低。
- 天然 TCR 药用化强度可能不足，且有脱靶风险。
- 仅靠结构预测或模型推断，仍难以可靠外推某个新 TCR 与某个 pMHC 是否真正形成有效且安全的结合；实验验证仍是核心环节。。

### 3. 计算侧可切入问题

- 抗体/识别头如何识别目标。
- 结合界面的结构基础。
- 不同表位为何带来不同效果。
- 亲和力优化与候选改造策略。
- 哪些问题可计算辅助，哪些必须湿实验验证。

### 4. 当前不该做的事

- 不要急着做大规模软件环境。
- 不要急着买算力。
- 不要把 AlphaFold 或单一结构工具误当成万能答案。
- 不要把“跑出结构”误解为“可直接做药”。

## 五、优先建立的常识地图（7 个词）

- **抗原**：可被免疫系统识别的目标。
- **MHC/HLA**：把小片段展示到细胞表面的“托盘”。
- **pMHC**：肽段 + MHC 的展示复合物。
- **TCR**：T 细胞表面的识别器。
- **BCR/抗体**：B 细胞体系中的识别器/分泌武器。
- **CAR-T**：工程化 T 细胞，偏表面靶点路线。
- **CoStAR**：工程化增强识别与激活能力的思路。

一句话串联：

> 肿瘤细胞把异常碎片贴到门口，T 细胞负责巡逻识别；工程化治疗要做的是把“识别异常 + 精准开火”做得更强、更准。

## 六、建议的启动路径

### 第一阶段：只学，不跑

先搞明白三个问题：

1. 为什么 CD70 是表面靶点，而 pMHC 是间接打胞内靶点。
2. 为什么天然 TCR 在药物化过程中会遇到强度、特异性和脱靶等问题。
3. 为什么“识别头”和“点火尾巴”都重要。

### 第二阶段：只读综述，不急复现

优先阅读方向：

- CD19 为什么成功。
- 不同表位为什么效果不同。
- CD70 已有哪些结构/抗体工作。
- pMHC 抗体为什么难、难在哪里。

### 第三阶段：再碰工具

等认知清晰后再进入：

- 结构可视化。
- 抗体/蛋白复合物分析。
- AlphaFold 与蛋白结构工具。
- 基础 docking / interface 分析。

否则容易变成“命令会敲，但不理解结果意义”。

## 七、当前待验证事项

+ CoStAR 具体分子构型、信号模块设计与适用边界，需要进一步查论文与综述确认。
+ pMHC 抗体路线在低密度靶点场景下的真实可转化性，需要结合更多文献与案例判断。
+ 当前可用的结构工具、商业许可与实际算力需求，后续再单独评估。
+ 计算侧能解决到哪一步、哪些环节必须依赖湿实验，后续需要逐项拆解。
