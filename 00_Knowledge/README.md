# 免疫治疗项目方向 沟通纪要

# 1. 目标

聚焦 **免疫治疗相关的具体靶点识别** 的 工程化问题, 不泛讨论 “AI+生物”

- 路线 A：`CD70`
  - 传统 `CAR-T` / `抗体工程` 思路。
  - 新的 表面靶点 `CD70`。
- 路线 B：`pMHC` / `CoStAR`
  - 识别肿瘤细胞表面的 `pMHC` 复合物。
  - 间接打到胞内突变抗原（`TP53` / `P53`）

# 2. 任务: 设计辅助 / 结构分析 / 候选筛选 / 知识整理

## 2.1 当前难点

- `pMHC` 靶点展示量低。
- 天然 `TCR` 药用化强度 可能不足，有脱靶风险。
- 仅靠 结构预测 / 模型推断，难以可靠外推 某个`TCR` / `pMHC` 是否有效 且 安全的结合；
- 实验验证仍是核心环节

## 2.2 计算侧 可切入 的 问题

- 抗体/识别头 如何 识别目标。
- 结合界面 的 结构基础。
- 不同表位 为何带来 不同效果。
- 亲和力优化 与 候选改造 策略。
- 哪些问题 可计算辅助，哪些必须 湿实验验证。

## 2.3 当前不该做

- 不要急着 做 大规模软件环境。
- 不要急着 买算力。
- 别把 单一结构工具 当 万能答案。
- 别把 “跑出结构” 当 “可直接做药”。

# 3、建议 路径

## 3.1 阶段1：只学，不跑

1. 为什么 CD70 是表面靶点，而 pMHC 是间接打胞内靶点。
2. 为什么天然 TCR 在药物化过程中会遇到强度、特异性和脱靶等问题。
3. 为什么“识别头”和“点火尾巴”都重要。

## 3.2 阶段2：只读综述，不急复现

- CD19 为什么成功。
- 不同表位为什么效果不同。
- CD70 已有哪些结构/抗体工作。
- pMHC 抗体为什么难、难在哪里。

## 3.3 阶段3：再碰工具

- 结构可视化。
- 抗体/蛋白复合物分析。
- AlphaFold 与蛋白结构工具。
- 基础 docking / interface 分析。
