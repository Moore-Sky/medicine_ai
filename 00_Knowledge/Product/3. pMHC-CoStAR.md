# 第三周：pMHC / CoStAR

# 第11天: 普通 `CAR-T` 够不着胞内抗原

- 普通抗体 / 普通 CAR-T 更擅长识别细胞表面完整蛋白，比如 CD19、CD70。
- 很多肿瘤相关蛋白在细胞里面，抗体和普通 CAR-T 的识别头通常进不去细胞内部，所以“看不见”这些靶点。
- 肿瘤细胞能把一部分胞内蛋白切成肽，装到 MHC-I/HLA-I 上，展示在细胞表面，这才给了 TCR / TCR-like antibody / pMHC-CAR 一类路线机会。

## 1. 细胞: 封闭房间

普通 CAR-T 的“眼睛”在细胞外; 细胞里面的东西，它正常情况下看不到。

- 细胞外部 -> 细胞膜 -> 细胞内部：蛋白、DNA、RNA、代谢物、突变蛋白……
- **普通抗体、普通 CAR-T 的识别头，基本都在细胞外面工作**。
  - 能看见：细胞表面露出来的蛋白, 比如：CD19、CD20、CD70、HER2、EGFR
  - 看不见：细胞里面的蛋白, 比如：p53、KRAS、MYC、很多突变蛋白

## 2. `p53` / `KRAS` 胞内蛋白

- p53 大部分时间在细胞里面工作; 不是像 CD19 那样插在细胞膜外面等着被抗体抓。

## 3. MHC-I / HLA-I 在这里像什么？

**MHC-I / HLA-I 像细胞表面的“样品展示架”或“安检窗口”**。

细胞会不断把自己内部的蛋白切成小片段，然后拿一部分放到 MHC-I 上，展示到细胞表面。

**胞内蛋白 → 被蛋白酶体切成短肽 → 短肽通过 TAP 进入内质网 → 装到 MHC-I / HLA-I 上 → 运到细胞表面 → 被 CD8 T 细胞 / TCR 检查**

简化版: 细胞内部材料 → 切片 → 装盘 → 摆到门口 → T 细胞检查

**不是只有肿瘤细胞会这么做。几乎所有有细胞核的人体细胞都会通过 MHC-I 展示内部蛋白片段**

- 正常细胞展示的大多是正常 self peptide；
- 肿瘤细胞如果有突变蛋白，可能会展示 mutant peptide，也就是`突变肽`。

## 4. 什么是 pMHC？

- pMHC 就是：peptide + MHC (短肽 + MHC)
- 人体: peptide-HLA 短肽 + HLA
- **pMHC = 细胞内部蛋白片段 + 展示它的 HLA 架子**

TP53 突变 → mutant p53 蛋白 → 被切成`突变短肽` → 装到 HLA-A 上 → 形成 p53 mutant peptide-HLA → 被 TCR / TCR-like antibody / pMHC-CAR 识别

类比: **CD70 是直接看服务端口；pMHC 是看一小段采样日志来判断内部有没有异常**。

- CD70/CD19 像 API endpoint：/cd19, /cd70; 可以直接请求它、识别它。
- pMHC 像系统内部日志被采样后，贴到 外部监控 面板:
  - 内部异常 protein mutation → 被采样成 peptide → 被 HLA 展示成 pMHC → 外部监控 TCR 读取
  - 不是所有异常 都会被 采样
  - 不是所有采样 都会 展示
  - 展示出来的日志 可能很少
  - **不同机器的日志格式还不一样，也就是 HLA 限制**

## 5. 为什么 普通抗体 / 普通 CAR-T 不直接进细胞？

抗体和 CAR-T 的识别模块，天然设计目标不是进入每个细胞内部搜查。

- 第一，细胞膜是屏障: **抗体是大蛋白，通常不能自由穿过完整细胞膜**。
- 第二，普通 CAR-T 的识别头长在 T 细胞表面
  - CAR-T 的 scFv 是伸在 T 细胞外面的, 只能跟另一个细胞表面暴露的抗原发生接触。
- 第三，细胞内部目标太多，不能乱抓
  - 如果免疫系统能随便进入每个细胞内部抓蛋白，那正常细胞也会被搞得很惨。

自然免疫系统: 细胞自己展示内部样品, T 细胞在外面检查样品

**普通 CAR-T 最擅长看标签。TCR/pMHC 路线是在看采样日志**

`MHC-I` / `HLA-I` 细胞把“内部蛋白样品”展示给免疫系统看的窗口。

T-细胞 在外面检查：

- 这个细胞展示出来的短肽正常吗？
- 有没有病毒肽？
- 有没有突变肿瘤肽？

## 6. pMHC 比 CD70 难？

CD70 路线：细胞表面有没有 CD70？有 → 识别; 没有 → 不识别

pMHC 路线：

- 细胞里有没有突变蛋白？
- 这个蛋白会不会被切出合适短肽？
- 这个短肽能不能被病人的 HLA 装上？
- 装上后能不能运到细胞表面？
- 表面数量够不够？
- TCR 会不会识别？
- 会不会误伤相似正常肽？

难点:

- 不是所有胞内蛋白都能被展示出来
- 不是所有短肽都能绑定 HLA
- 不同病人 HLA 不一样
- pMHC 表面密度可能很低
- TCR 可能误认相似短肽
- 结构预测不能完全替代实验验证

## 7. 小科普

- MHC 动物, HLA 人体
- MHC-I / HLA-I：展示细胞内部蛋白片段，主要给 `CD8` T 细胞看。
- MHC-II / HLA-II：展示吞进来的外部蛋白片段，主要给 `CD4` T 细胞看。
- HLA-I 里面还有 HLA-A、HLA-B、HLA-C。
- 具体到药物靶点时，经常要写成 HLA-A\*02:01 这种具体型号。

# 第12天：p53 / TP53、突变短肽、表面呈递

## 1. 分清 `TP53` 和 `p53`

- `TP53` = 基因
  - TP53 mutation = TP53 基因发生突变
- `p53`= 这个基因编码出来的蛋白
  - mutant p53 protein = 突变后的 p53 蛋白

详细:

- `p53`: 肿瘤抑制蛋白，常被叫做 “guardian of the genome”。
- `TP53`: 人癌症里最常见的突变基因之一，一直是很诱人的癌症治疗靶点；
  - 主要是胞内蛋白，很多肿瘤相关抗原位于细胞内部，因此普通抗体类疗法不能直接接触这些靶点。

## 2. 不是 “打 p53”，是 “打 p53 的展示片段”

**真正可被免疫识别的不是 整段 p53 蛋白, 而是：某个 p53-derived peptide + 某个 HLA**

误区: 肿瘤细胞有 mutant p53 → 做一个药识别 mutant p53 → 杀掉肿瘤

准确： TP53 基因突变→ 产生 mutant p53 蛋白 → mutant p53 蛋白被细胞降解 → 产生一些短肽 → 某些短肽被特定 HLA-I 装载 → 形成 peptide-HLA / pMHC → 展示到细胞表面 → TCR / TCR-like antibody / pMHC-CAR / Co-STAR 去识别

## 3. 为什么 p53 很诱人？

- TP53 突变非常常见
  - p53 hotspot mutations 可能有免疫原性，并能引发针对 mutant p53 neoantigens 的肿瘤内 T 细胞反应。
- 很多癌种都可能有 TP53 突变
  - 肺癌、结直肠癌、卵巢癌、头颈癌等，都可能出现 TP53 突变。
- 它是癌症核心通路
  - p53 不是边角料蛋白; 它参与 DNA 损伤应答、细胞周期、凋亡、衰老等关键过程。

## 4. p53 难点

- TP53 有没有突变？
- 突变蛋白有没有表达？
- 蛋白有没有被降解成合适短肽？
- 短肽能不能被该病人的 HLA 装载？
- pMHC 有没有运到细胞表面？
- 表面密度够不够？
- 识别模块够不够特异？
- 会不会误伤正常 pMHC？

## 5. `突变短肽`

- p53 蛋白某个位点突变了。
- `R175H`
  - R175H = 第 175 位氨基酸从 R 变成 H
  - R = arginine，精氨酸
  - H = histidine，组氨酸
  - 突变后，细胞里的 p53 蛋白会被处理系统切成一些短肽。

坑：突变 -- 合适短肽 -- 能绑定 HLA -- 展示量够 -- 安全可打

- 有突变 ≠ 产生合适短肽
- 有短肽 ≠ 能绑定 HLA
- 能绑定 HLA ≠ 展示量够
- 展示量够 ≠ 安全可打

## 6. HLA 限制性：为什么不是所有病人都能用？

同一个 p53 突变短肽，不是所有 HLA 都能展示。

方案靶向 `p53 R175H + HLA-A*02:01`, 同时满足：肿瘤有 TP53 R175H 突变; 病人有 HLA-A\*02:01

- 如 病人是 TP53 R175H 突变,但 HLA 不是 A\*02:01, 那这个 pMHC 靶点可能根本不存在。
- 如 病人有 HLA-A\*02:01, 但没有 TP53 R175H, 也没用。

**pMHC 靶点 二元组：突变位点 + HLA 型号**

案例：**TP53 R175H + HLA-A**

研究 TP53 突变 R175H，并识别这个突变形成的 p53 peptide-HLA 复合物。

Johns Hopkins 的摘要说，这项工作识别出一种抗体，能高度特异地识别 TP53 R175H 突变与常见 HLA-A 等位基因形成的细胞表面复合物，并把它改造成 bispecific single-chain diabody；

即使癌细胞表面的 p53 peptide-HLA 密度极低，这个双特异性分子仍能在体外和小鼠实验中激活 T 细胞杀伤呈递该新抗原的癌细胞。

- 胞内突变蛋白, 真的可以通过 pMHC 变成表面靶点
- 同时也看到：这个表面靶点非常低密度, 而且高度依赖突变位点和 HLA 型号
- 太像做低信噪比监控：日志有，但少得离谱。

## 7. TCR-like antibody / TCRm antibody 是什么？

pMHC 路线不总是“只看突变氨基酸本身”，有时也可能利用肿瘤细胞中特定蛋白的异常积累和展示量差异。

Nature Communications 2019 工程化出 P1C1TM，识别 p53 125–134 peptide + HLA-A24:02 这个 pMHC。

- 肿瘤细胞表达 mutant p53 时，p53-derived pMHC 水平可能更高，从而让这个 TCR-like 抗体能区分某些 mutant p53 表达细胞和 wild-type p53 表达细胞。
- 识别的是 wild-type p53 125–134 peptide + HLA-A24:02，但 mutant p53 肿瘤细胞里 p53 蛋白积累更高，导致这个 p53-derived pMHC 展示水平更高，所以可以利用“展示量差异”来区分。

# 第13天：天然 TCR 的限制

## 1. 目标

胞内蛋白 → 被切成短肽 → 装到 MHC-I / HLA-I → 形成 pMHC → 被 TCR 检查

最后一步：

- TCR 怎么判断这个 pMHC 是不是该打？
- TCR 看的不是一个完整蛋白，而是一个非常小的结构组合：

**pMHC = 短肽 peptide + HLA/MHC 架子**

- TCR 识别立体表面, TCR 与 pMHC 结合强度、结构接触、动力学特征 都影响 T-细胞 反应
- affinity、specificity、cross-reactivity 放在一起理解

自然免疫里的 TCR: 有限 TCR 数量 -> 需要覆盖巨大 peptide 空间 -> 天然允许一定 cross-reactivity -> 好处：覆盖未知病原体

工程 TCR 治疗：目标是杀肿瘤 --> 必须避免杀正常组织 --> cross-reactivity 变成脱靶风险 --> 需要大量实验验证

## 2. 术语

#### 2.1 affinity `亲和力`: TCR 和某个 pMHC 绑得有多紧

描述：

- `KD`：整体 亲和力
- `kon`：结合 速度
- `koff`：解离 速度

注意:

- 亲和力高 ≠ 一定疗效好
- 亲和力低 ≠ 一定没反应

#### 2.2 specificity `特异性`: TCR 是不是主要只认目标 pMHC？

例子: TP53 R175H mutant peptide + HLA-A\*02:01

理想情况是：

- 只认这个 mutant pMHC
- 不认 正常 p53 肽
- 不认 其他正常人体肽
- 不认 其他相似 peptide-HLA

现实: 很难做到绝对“只认一个”。

- TCR 识别 三维接触面。
- 不同短肽，只要在关键位置上长得像，就可能被同一个 TCR 部分识别。

#### 2.3 cross-reactivity `交叉反应`: TCR 除了识别目标 pMHC,还可能识别其他相似 pMHC。

- cross-reactivity = 覆盖面更大
- 工程治疗：cross-reactivity = 可能误伤正常组织

#### 2.4. 关系

**工程治疗最难的是同时满足：强 / 准 / 不误伤**

- affinity: 绑得有多强 --> 影响激活概率 --> 但不等于 specificity
- specificity: 是不是主要认目标 --> 决定治疗是否精准 --> 受 cross-reactivity 影响
- cross-reactivity: 会不会认相似目标 --> 自然免疫需要它 --> 工程治疗害怕它

## 3. 为什么天然免疫需要交叉识别？

- TCR 的数量虽然很多，但潜在 peptide-HLA 组合数量更恐怖。
- 自然 策略：每个 TCR 不是只认一个目标，而是能覆盖一小片相似结构空间。

好处：

- 覆盖未知病原体
  - 模糊识别可以让 T 细胞对一组相似抗原仍有反应。
- 用有限 TCR 覆盖巨大抗原空间
  - “压缩编码: 一个 TCR 能识别多个相关结构

坏处:

- 泛化能力的反面就是误判。

工程治疗有风险:

- 工程 TCR-T / pMHC-CAR / Co-STAR 的目标: 不是“泛泛防御”，而是 杀肿瘤, 不要杀正常组织
- 自然免疫：宁可模糊一点，提高覆盖率
- 工程治疗：必须非常谨慎，因为**误杀正常组织代价很高**

## 4. 误区: 亲和力越高越好

- 调高亲和力时，可能不只是增强目标识别，也会增强对相似正常 pMHC 的识别。
- 亲和力增强 TCR 虽然可能提高治疗活性，但 off-target cross-reactivity 是一个重要毒性风险。

理解:

- 低亲和力 TCR：目标识别弱，但误伤也可能低
- 高亲和力 TCR：目标识别强，但可能把相似正常肽也拉进激活范围

优化: `therapeutic window` `治疗窗口`

- 目标 pMHC 反应强
- 非目标 pMHC 反应弱
- 正常组织不反应

## 5. TCR 和 pMHC 是怎么结合的？

问题: 如果另一个正常 peptide 长得和目标 peptide 在关键接触位点很像，那么 TCR 可能也会识别。

TCR 像一只手, pMHC 像一个托盘上放着一小段样品

```txt
        TCR
         ↓
┌───────────┐
│ peptide │ ← 短肽，真正携带“异常信息”
┌──┴───────────┴──┐
│ HLA/MHC │ ← 展示架
└─────────────────┘
```

## 6. 边界: 计算模型还不能直接替代实验？

**计算可以帮助筛选、排序、排雷; 但不能直接宣布安全有效**

计算模型要同时理解：TCR 序列, peptide 序列, HLA 类型, 三维结构, 结合动力学, 细胞激活阈值, 抗原密度, 正常组织表达

很多模型 尝试预测 TCR specificity / cross-reactivity，临床应用需要更好地理解 TCR:pMHC 相互作用，并结合实验与计算方法。

## 7. TCR / pMHC 项目 直觉

不要只问：这个 TCR 亲和力高不高？

而要问：

- 它识别哪个 peptide-HLA？
- 目标 peptide 在肿瘤上展示密度够不够？
- 正常组织有没有相似 peptide-HLA？
- 这个 TCR 有没有做 cross-reactivity screening？
- affinity 是天然的，还是人工增强过？
- 人工增强后 specificity 有没有变差？
- 有没有正常细胞杀伤实验？

# 第14天：亲和力、特异性、脱靶

## 1. pMHC 靶点: 某个突变短肽 + 某个 HLA 型号

pMHC = peptide + MHC/HLA 短肽 + 展示架

- TP53 R175H mutant peptide + HLA-A\*02:01
- KRAS G12D mutant peptide + HLA-C\*08:02

比 CD70 更“窄”、更“小”、更依赖上下文

## 2. 为什么 pMHC 比 CD70 难？先看 CD70

CD70: 肿瘤细胞表面有 CD70 --> 抗体 / CAR-T 识别 CD70 胞外表位 --> 结合 --> 激活 / 杀伤

pMHC：细胞内部有突变蛋白 --> 突变蛋白被切成短肽 --> 短肽被某个 HLA 装载 --> pMHC 被运到细胞表面 --> 表面数量可能很少 --> TCR / TCR-like antibody 识别 --> T 细胞激活杀伤

#### 2.1. 目标：识别要更强

pMHC 问题：目标太小、太少、太像正常东西。

- CD70 是一个完整表面蛋白上的胞外表位。
- pMHC 是一个短肽加 HLA 形成的小型复合表面。

从一堆正常 pMHC 里找出一个异常 pMHC：

```txt
正常 peptide-HLA
正常 peptide-HLA
正常 peptide-HLA
突变 peptide-HLA  ← 目标
正常 peptide-HLA
正常 peptide-HLA
```

- 识别要更强：对目标 pMHC 结合更强, 对目标 pMHC 更敏感, 目标数量很少也能看见
- 问题：识别更强，不一定更安全。

#### 2.2. 目标：点火要更敏感

pMHC 问题：表面密度可能很低。

癌细胞表面可能只有很少几个目标 pMHC; 那 T 细胞如果太“迟钝”

就会发生：

- 癌细胞：我表面有 5 个目标 pMHC
- T 细胞：太少了，没感觉
- 癌细胞：溜了

点火要更敏感: 更低抗原密度也能激活, 更少 pMHC 也能杀伤, 更弱信号也能被放大
问题：点火太敏感，就可能被正常细胞上的弱相似信号点燃。

#### 2.3. 目标：脱靶要可控

pMHC 最大的安全难点之一是：相似 self-peptide + HLA 可能被误认。

- 一个 TCR 可能不是只识别目标 pMHC。
- TCR 天然具有交叉识别能力，因为人体 TCR 数量有限，而潜在 peptide-HLA 空间巨大；计算预测 cross-reactivity 的综述也强调，TCR 识别 pMHC 既有高度特异性，又天然存在交叉反应这一面。

矛盾:

- 自然免疫 -- 优点：有限 TCR, 覆盖大量未知病原体, 需要一定模糊识别
- 治疗 -- 风险：相似 self-peptide + HLA 可能被误认, 导致正常细胞被攻击

pMHC 项目不能只问：能不能杀肿瘤？还要关注 **脱靶风险：会不会杀正常组织**

#### 2.4. 矛盾

- 亲和力调高, 识别更强, 但可能同时让一些相似正常 pMHC 也被识别. 疗效增强,但脱靶风险上升
- 点火更敏感，T-细胞激活阈值调低, 低密度目标 pMHC 也能激活, 但正常细胞上低水平相似 pMHC 也可能激活
- 脱靶要可控，又会限制疗效; 如果你为了安全，把识别和点火都调得很保守：

## 3. 例子: TP53 R175H + HLA-A\*02:01

TP53 R175H mutant peptide + HLA-A\*02:01

第一步：病人有没有突变 TP53 R175H mutation

没有，就没这个目标。

第二步：病人有没有 HLA-A\*02:01?

没有，也没这个目标。

第三步：肿瘤细胞 有没有真的展示？

有突变不等于一定展示。

mutant p53 protein → mutant peptide → HLA-A\*02:01 → cell surface pMHC

第四步：展示密度够不够？

- 表面 pMHC 很多 → 容易识别
- 表面 pMHC 很少 → 需要更敏感的识别/点火

第五步：TCR 会不会认错？要排查：

- 有没有其他正常 peptide + HLA-A\*02:01 长得像？
- 有没有正常组织表达这些 peptide 来源蛋白？
- TCR 会不会被它们激活？

## 4. 类比：pMHC 是低频危险日志

- CD70：服务器 暴露 endpoint：/cd70; 药物可以直接扫：发现 /cd70 → 触发处理
- pMHC：系统内部有 bug：TP53_R175H; 但外部看不到内部模块。系统只会偶尔把内部状态采样成日志：[HLA-A*02:01] TP53_R175H_peptide

问题：

- 不是所有 bug 都会打日志
- 不同机器日志格式不同，HLA 不同
- 这条日志可能每小时只出现几次
- 正常日志里可能有相似格式
- 报警阈值太低会误报，太高会漏报

# 第15天：Co-STAR / CoStAR

## 1. 背景

如果目标 pMHC 太少，普通 TCR 或普通 CAR 都不够理想，那能不能造一个混合系统，让 T 细胞对低密度 pMHC 更有反应？

Co-STAR 不是主线基础知识，是一个“工程方案案例”。

## 2. Co-STAR 想解决什么问题？

**p53 R175H peptide + HLA-A2**

这个目标来自细胞内部的 mutant p53。它是：突变蛋白 → 短肽 → HLA-A2 展示 → peptide-HLA / pMHC

- 这种 pMHC 在癌细胞表面的数量可能非常少。
- 靶向的是 p53 R175H 突变肽展示在 HLA-A2 上的 peptide-HLA 抗原，而且这类抗原在癌细胞表面可能只有 1–10 个；经典 CAR 格式难以对这么少的抗原产生反应。

**Co-STAR 的目标：为低密度 peptide-HLA 靶点增强识别和点火**。

## 3. Co-STAR 设计

- 识别层：TCR + antibody-like recognition
- 信号层：TCR signaling + MyD88/CD40 等增强模块
- 目标层：低密度 peptide-HLA，比如 p53 R175H-HLA-A2

Co-STAR 组 来自几类免疫细胞的组件：T 细胞的 TCR、B 细胞的抗体、单核细胞相关的 MyD88、树突细胞等相关的 CD40；其中 TCR 和抗体部分像“入侵者探测器”，MyD88 和 CD40 负责增强这个“报警”信号。

理解：

- TCR：适合读 pMHC
- 抗体模块：提供工程化识别能力
- MyD88/CD40：提供更强的信号放大

**解决：目标 pMHC 太少 -> 普通 CAR/TCR 不够强 -> 加入混合识别 + 信号增强 -> 让 T 细胞更容易对低密度 pMHC 产生持续反应**

## 4. Co-STAR 对比 普通 CAR-T

普通 CAR-T 更像“看到明显门牌就攻击”；Co-STAR 更像“读极低频内部异常日志，并配一个更强报警放大器”。

- 普通 CAR-T： 抗体识别头 + T-细胞 激活尾巴; 适合 细胞表面蛋白表位
- Co-STAR：TCR/pMHC 识别能力 + 抗体工程模块 + 额外信号增强模块, 适合低密度 peptide-HLA / pMHC

## 5. Co-STAR 对比 TCR-T

- TCR-T：给 T 细胞装一个特定 TCR → 识别 peptide-HLA → 激活杀伤, 天然适合看 pMHC; 但低密度 pMHC、信号不足、持久性、脱靶等仍然很麻烦。
- Co-STAR：不只给一个 TCR, 而是把 TCR/CAR 的优点和额外信号模块组合起来

## 6. Co-STAR 和 CoStAR

CoStAR: 给 TIL/T 细胞补共刺激。单独不会激活 T 细胞，而是与 TCR 信号结合后增强 T 细胞活性、细胞因子分泌和细胞毒性。

| CoStAR  | costimulatory antigen receptor，偏“给已有 TCR/TIL 补共刺激” |
